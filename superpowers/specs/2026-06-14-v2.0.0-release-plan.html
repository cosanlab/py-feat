py-feat v2.0.0 release plan#

Date: 2026-06-14 Status: In progress — safe prep landed on chore/v2.0.0-release-prep Author: Luke Chang (with Claude Code) Related: 2026-06-03-docs-modernization-marimo-book.md (docs strategy), 2026-06-14-pyfeatlive-dispatch-setup.md (changelog automation), CLAUDE.md (branch/version policy)

Why 2.0.0 (not 0.7.0)#

v0.7-dev is a major structural overhaul + a coordinated batch of breaking changes (Python 3.11+, dependency removals, Detector→Detectorv1 rename, the new Detectorv2 multitask model, ArcFace identity default, modern model formats). It also debuts the relaunched marimo-book docs and the pyfeat-live desktop app. That’s a major version. We ship the whole v0.7-dev line to PyPI as 2.0.0.

0.7.0 was only ever a pre-release git tag for testers (pip install git+...@v0.7.0). 2.0.0 supersedes it. Per CLAUDE.md, do not re-tag v0.7.0 — testers’ URLs must keep resolving.

Scope decisions (locked 2026-06-14)#

  • Docs: parallel preview. The marimo-book site (docs/book.yml) builds in CI and can publish to py-feat.org/preview/; the live py-feat.org stays on jupyter-book for the 2.0.0 cut. Full cutover is a fast-follow.

  • pyfeat-live: repo is public → py-feat’s changelog refresh uses the default token; the only new credential is a dispatch PAT inside pyfeat-live.

  • Version jump: 0.7.0 → 2.0.0 (skip 1.x).

Audit results (feat/ package)#

Full report from the dead-code/secret audit:

  • Secrets/credentials: none. No hf_/ghp_/AKIA tokens, no hardcoded passwords, no baked absolute paths. .env is untracked + gitignored.

  • No import-time blockers, no stale Detector class references (rename is clean), no breakpoint()/pdb.

  • Fixed: the two f-string bugs above.

  • Deferred (nice-to-have, not blockers): commented-out dead code in feat/identity_detectors/facenet/facenet_model.py (lines 1-13, 381-417) and small remnants in image_operations.py, plotting.py, a few detector model files; a non-verbose-gated print at feat/data.py:2797; open TODO/FIXME markers (known limitations). Clean these in a follow-up sweep, not in the release-prep PR.
