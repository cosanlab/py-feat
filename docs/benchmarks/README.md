# py-feat detector benchmarks

This directory tracks reproducible benchmark runs of `Detector` and
`MPDetector` over time. Each file is a single run, named
`YYYY-MM-DD-<git-sha>.md`, generated by `scripts/bench_detectors.py
--markdown`.

## Generating a new run

```bash
# Default: device x batch sweep, write markdown alongside stdout
python scripts/bench_detectors.py --markdown

# Add the num_workers axis (slower; doubles runtime)
python scripts/bench_detectors.py --workers 0 2 --markdown

# Custom batch sizes and a single device
python scripts/bench_detectors.py --batches 1 8 32 --devices mps --markdown

# Skip the long-video sweep (cuts runtime ~5x)
python scripts/bench_detectors.py --skip-long-video --markdown
```

Each run records hardware, Python / PyTorch / py-feat / git versions, the
swept axes, and per-cell wall times in markdown tables. Designed to
make perf regressions easy to spot across releases.

## Reading the files

Every benchmark file has two parts:

1. **Script-generated** (always present, produced by `--markdown`):
   - Run metadata block — date, py-feat version, git commit, host, Python / PyTorch versions, OMP_NUM_THREADS, swept axes
   - One section per dataset (video, images), each with one subsection per configuration (`img2pose`, `retinaface`, `MPDetector retinaface`)
   - Each subsection is a markdown table with one row per (device × batch × workers) cell

2. **Optional hand-curated narrative** (appended below the script-generated part with a `---` separator):
   - Headline numbers — the best ms/frame and key cross-device comparisons
   - Findings & recommendations — interpretation of what the numbers say

The hand-curated narrative is optional. It's worth adding for runs that change the recommended defaults (e.g., the inaugural `2026-05-03-*.md` after the OMP_NUM_THREADS fix); routine regression-tracking runs don't need it.

## What's in scope

`scripts/bench_detectors.py` measures the **detection + landmark + AU
only** path (emotion / identity / facepose disabled). This is to keep
the swept axes (face_model x device x batch_size) as the only variables.
For full-stack benchmarks (with resmasknet emotion, ArcFace identity,
PnP pose, etc.), write a separate one-shot script — those depend on
which downstream stages a given user enables.
