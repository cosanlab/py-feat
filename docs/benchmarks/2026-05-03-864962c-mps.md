## Notes (hand-curated)

**Scope:** MPS-only sweep across the three detector configurations at batch sizes `[1, 4, 16]`, `workers=0`. CPU device skipped (the v0.7 perf changes don't materially affect CPU; the previous baseline `2026-05-03-437b651.md` covers the CPU + workers axis).

**Headline numbers (best ms/frame at batch=16, workers=0, MPS):**

| MPDetector(face_model='retinaface', au_model='mp_blendshapes') | 17.5 ms/frame (57.1 fps) | 54.5 ms/img |

**HOG batching speedup (PR #292, isolated bench):**

The detector-level numbers above include face detection + landmark + AU, which dilutes the HOG speedup measurement. Isolated `extract_hog_features_batched` vs the legacy `extract_hog_features` per-face loop at increasing face counts on MPS:

| n_faces | Legacy | Batched (PR #292) | Speedup |
|---|---|---|---|
| 5 | 15.2 ms | 7.8 ms | 1.96x |
| 20 | 68.1 ms | 16.3 ms | 4.19x |
| 50 | 145.9 ms | 32.0 ms | 4.56x |

The win comes from removing the per-face GPU↔CPU sync the legacy loop forced (~98% of MPS wall time). HOG output is parity-preserved (mean abs diff < 5e-6 vs legacy; AU values bit-identical end-to-end on test images). See PR #292 for details.

**Comparison to prior baseline (`2026-05-03-437b651.md`):** the prior file is hand-curated from a different scope (retinaface only, workers=[0,2] sweep) and predates several v0.7 changes including the bench script extension in PR #289. Direct cell-by-cell comparison conflates scope and methodology differences. This file is the canonical post-PR-292 MPS reference; future runs should compare against it.
